# نظام إدارة البنك (Bank System - v1.0)

يُعد هذا المشروع تطبيقاً برمجياً متكاملاً يعمل عبر واجهة سطر الأوامر (CLI)، تم تطويره بالكامل باستخدام لغة C++. يهدف النظام إلى إدارة بيانات عملاء البنك بكفاءة عالية، ويُعتبر تتويجاً عملياً لدمج مفاهيم البرمجة المتقدمة، بما في ذلك تصميم الخوارزميات، توظيف هياكل البيانات (Data Structures)، وإدارة الملفات النصية كقاعدة بيانات محلية (Flat-File Database).

---

## المميزات الحالية (المرحلة الأولى: الأساسيات وإدارة السجلات)

تم الانتهاء من بناء المحرك الأساسي لإدارة السجلات (Client Management Core)، والذي يدعم عمليات (CRUD) المتكاملة وفقاً لأفضل الممارسات الهندسية:

- **استعراض قائمة العملاء (Read):**
  - جلب كافة بيانات العملاء من قاعدة البيانات وعرضها في جدول ديناميكي منسق بعناية باستخدام مكتبة `<iomanip>`.

- **إضافة عميل جديد (Create):**
  - تطبيق نظام تحقق صارم لمنع تكرار "رقم الحساب" (Unique Account Number Validation).
  - الاعتماد على تقنية الإضافة المزدوجة والمباشرة في الذاكرة العشوائية والملف النصي لتحقيق تعقيد زمني مثالي (O(1) Optimization) دون الحاجة لإعادة تحميل الملف بالكامل.

- **حذف سجل عميل (Delete):**
  - تطبيق خوارزمية الحذف الآمن باستخدام وسم السجلات (Soft Delete / Tombstone Pattern)، يليه عملية إعادة كتابة شاملة للملف (Overwrite) لضمان استقرار الفهارس والبيانات.

- **تحديث بيانات العميل (Update):**
  - توفير واجهة لتعديل بيانات العميل بشكل فردي (الرمز السري، الاسم، رقم الهاتف، ورصيد الحساب) مع مزامنة التغييرات مباشرة في الذاكرة والقرص الصلب.

- **البحث المتقدم (Find):**
  - محرك بحث يعتمد على خوارزمية البحث الخطي (Linear Search) للوصول السريع إلى أي عميل باستخدام رقم الحساب وعرض بطاقة بياناته المستقلة.

- **حماية البيانات والتنقل الآمن:**
  - استبدال الاستدعاء الذاتي (Recursion) بحلقات التكرار `while loop` لإدارة القوائم، مما يمنع حدوث طفح الذاكرة (Stack Overflow).
  - بناء طبقة حماية لمعالجة أخطاء إدخال المستخدم (Input Validation) لمنع انهيار البرنامج أثناء التنفيذ.

## التقنيات وهياكل البيانات المستخدمة

- **لغة البرمجة:** C++

- **هياكل البيانات:**
  - `struct`: لبناء كائن موحد يمثل بطاقة بيانات العميل.
  - `std::vector`: لإدارة مجموعة العملاء ديناميكياً داخل الذاكرة العشوائية (RAM) وضمان سرعة المعالجة.

- **قاعدة البيانات:**
  - ملف نصي `Clients.txt` يمثل قاعدة البيانات (Flat-File).
  - بناء نظام تجزئة نصي (Parsing Engine) يعتمد على فاصل مخصص `"#//#"` لدمج وفصل الحقول برمجياً.

- **المنطق المعماري:**
  - الاعتماد على البرمجة التركيبية (Modular Programming) لفصل واجهة المستخدم (UI) تماماً عن العمليات المنطقية لمعالجة البيانات (Business Logic).

## خارطة الطريق (المرحلة الثانية: العمليات المالية)

يجري العمل حالياً على توسيع معمارية النظام ليشمل قسماً مستقلاً خاصاً بالمعاملات المالية (Transactions)، والذي سيتضمن:

- **شاشة المعاملات (Transactions Menu):** بناء قائمة فرعية معزولة بالكامل عن قائمة إدارة السجلات الرئيسية.
- **الإيداع (Deposit):** نظام لإضافة مبالغ مالية لحسابات العملاء مع مزامنة فورية للرصيد في الملف.
- **السحب (Withdraw):** نظام خصم مالي يتضمن طبقة تحقق مسبقة لضمان كفاية الرصيد (Balance Validation).
- **إجمالي الأرصدة (Total Balances):** وحدة تقارير مالية تقوم بحساب وعرض المجموع الكلي لأموال جميع المودعين داخل البنك.
- **الربط الديناميكي:** هندسة انتقال سلس وآمن للعودة والتنقل بين القائمة الرئيسية وقائمة العمليات المالية دون فقدان حالة البرنامج (State).
