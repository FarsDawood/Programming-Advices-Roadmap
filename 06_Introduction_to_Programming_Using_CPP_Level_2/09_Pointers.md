# الفصل التاسع: المؤشرات (Pointers)

## 🎯 الدرس الخامس والثلاثون: ما هو المؤشر؟ (What is Pointer?)

موضوع المؤشرات (Pointers) هو "البعبع" الذي يخيف الكثير من دارسي لغة C++، ولكنه في الحقيقة موضوع في غاية السهولة والمنطقية إذا فهمت المفهوم (Concept) الأساسي خلفه.

لغة C++ هي من اللغات النادرة التي تسمح لك باستخدام المؤشرات صراحةً. اللغات الحديثة مثل C# و Java لا تسمح لك بذلك بشكل مباشر (تقوم بإدارتها في الخلفية)، وهذا ما يعطي لـ C++ قوتها الجبارة: التحكم الكامل والمباشر في الذاكرة (Full Memory Control).

### 1. مراجعة سريعة للمتغير العادي (Normal Variable):

عندما نكتب int a = 10;، يتم إنشاء "صندوق" في الذاكرة يحتوي على 3 أشياء:

الاسم: a

القيمة: 10

العنوان في الذاكرة (Address): مثل 0x7ffeeb42

### 2. ما هو المؤشر (Pointer)؟

المؤشر بكل بساطة هو متغير عادي جداً، ولكنه لا يُخزن أرقاماً (مثل 10 أو 20) ولا نصوصاً، بل يُخزن "عناوين" (Addresses) لمتغيرات أخرى أو دوال أو كائنات في الذاكرة!

وظيفة المؤشر هي "التأشير" على مكان معين في الذاكرة لتتمكن من الوصول إليه لاحقاً.

### 3. التشبيه الهندسي العبقري (صندوق الـ 2 طن):

تخيل أن لديك صندوقاً وزنه 2 طن مليء بالبضائع (مثال لمتغير ضخم في الذاكرة)، وتريد تسليمه لشخص آخر.

الطريقة التقليدية (بدون مؤشر): ستقوم بحمل الصندوق (2 طن) ونقله ونسخه للشخص الآخر (عملية بطيئة ومستهلكة للموارد).

طريقة المؤشرات (Pointers): لن تحرك الصندوق من مكانه أبداً! بل ستعطي الشخص ورقة صغيرة مكتوب عليها "عنوان" المخزن ومعه المفتاح ليتعامل هو مع الصندوق مباشرة. هذه الورقة الصغيرة هي الـ Pointer.

### 4. طريقة التعريف والتهيئة (Declaration & Initialization):

لتعريف مؤشر، نستخدم علامة النجمة \* (Asterisk) قبل اسم المتغير، ولجلب عنوان متغير آخر نستخدم علامة & (Ampersand).

⚠️ قاعدة ذهبية: يجب أن يكون نوع المؤشر هو نفس نوع البيانات للمتغير الذي يشير إليه. (إذا كان المتغير int، يجب أن يكون المؤشر int \*).

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

int main()
{
    // 1. تعريف متغير عادي
    int a = 10;

    // 2. تعريف المؤشر (استخدمنا * للدلالة على أنه مؤشر)
    // وقمنا بتخزين "عنوان" a بداخله باستخدام &
    int *p = &a;

    // طباعة القيمة والعنوان من المتغير الأصلي
    cout << "Value of a: " << a << endl;             // النتيجة: 10
    cout << "Address of a (&a): " << &a << endl;     // النتيجة: عنوان سداسي عشر (مثال: 00A7F85C)

    // طباعة ما يحتويه المؤشر (سنجد أنه يحتوي على عنوان a)
    cout << "\nValue inside Pointer p: " << p << endl; // النتيجة: نفس العنوان السداسي عشر (00A7F85C)

    return 0;
}
```

</div>

### 5. مرونة المؤشرات (Dynamic Pointing):

من أجمل مميزات المؤشر أنه "مؤشر حر" أثناء عمل البرنامج (Runtime). يمكنك في السطر الأول أن تجعله يشير إلى المتغير a، وفي السطر الذي يليه تجعله يغير رأيه ويشير إلى عنوان المتغير b.

<div dir="ltr">

```cpp
int a = 10;
int b = 20;

int *p; // تعريف المؤشر فقط

p = &a; // الآن المؤشر يشير إلى عنوان a
p = &b; // الآن المؤشر غير اتجاهه وأصبح يشير إلى عنوان b

```

</div>

💡 الخلاصة المبدئية: المؤشر هو مجرد "دفتر عناوين". يخزن عنوان الذاكرة لشيء آخر حتى نتمكن من الوصول لهذا الشيء بسرعة فائقة ودون الحاجة لنسخه.

## 🔑 الدرس السادس والثلاثون: فك المرجعية للمؤشرات (Dereferencing Pointers)

في الدرس السابق، تعلمنا كيف نجعل المؤشر "يؤشر" (Reference) على متغير آخر عن طريق تخزين عنوانه.
في هذا الدرس سنتعلم العكس: كيف نستخدم هذا العنوان للوصول إلى القيمة الحقيقية الموجودة بداخله والتعديل عليها؟ هذه العملية تُسمى فك المرجعية (Dereferencing).

### 1. مفهوم الـ Dereferencing (مفتاح الصندوق):

تخيل أن المؤشر p هو ورقة مكتوب عليها عنوان صندوق في مستودع ضخم (الذاكرة).

إذا طبعت p مباشرة، سيطبع لك "العنوان" المكتوب على الورقة.

ولكن، ماذا لو أردت فتح الصندوق لمعرفة ما بداخله أو لتغيير محتواه؟ هنا تحتاج إلى "مفتاح".

في لغة C++، المفتاح هو علامة النجمة * قبل اسم المؤشر (مثال: *p).

### 2. التعديل المزدوج (Two Ways, One Box):

بما أن المتغير الأصلي a والمؤشر p متصلان بنفس الصندوق (العنوان) في الذاكرة، فإن:

أي تعديل تقوم به باستخدام المتغير a، سينعكس فوراً على المؤشر p.

أي تعديل تقوم به باستخدام "مفتاح" المؤشر \*p، سينعكس فوراً على المتغير a.

💻 تطبيق عملي شامل بالكود:

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

int main()
{
    int a = 10;
    int *p = &a; // Reference: المؤشر الآن يحمل عنوان المتغير a

    cout << "--- Initial Values ---\n";
    cout << "Value of a: " << a << endl;             // 10
    cout << "Value of Pointer (p): " << p << endl;   // Address (e.g., 0x7ffeeb42)

    // Dereferencing: استخدام المفتاح لجلب القيمة من داخل الصندوق
    cout << "Value pointed to by (*p): " << *p << endl; // 10

    cout << "\n--- Modifying via Pointer ---\n";
    *p = 20; // استخدمنا المفتاح لفتح الصندوق وتغيير قيمته إلى 20
    cout << "New Value of a: " << a << endl;         // 20
    cout << "New Value of (*p): " << *p << endl;     // 20

    cout << "\n--- Modifying via Variable ---\n";
    a = 30; // غيرنا القيمة من المتغير الأصلي
    cout << "New Value of a: " << a << endl;         // 30
    cout << "New Value of (*p): " << *p <<div endl;     // 30

    return 0;
}

```

</div>

🚨 تحذير هندسي هام: حاول دائماً التفرقة بين p و *p. إذا كتبت p = 20; بدون النجمة (المفتاح)، فإن الكومبايلر سيغضب ويعطيك خطأ (Error)، لأنك تحاول تخزين رقم عادي داخل مؤشر لا يقبل سوى "عناوين" الذاكرة! لتغيير القيمة، يجب استخدام المفتاح *p = 20;.

## 🚫 الدرس السابع والثلاثون: أخطاء شائعة مع المؤشرات (Common Mistakes with Pointers)

بما أن المؤشرات (Pointers) تعطي المبرمج صلاحيات مطلقة للتحكم في الذاكرة، فإن ارتكاب أخطاء بسيطة أثناء كتابتها قد يؤدي إلى غضب الكومبايلر أو انهيار البرنامج تماماً.

في هذا الدرس، سنستعرض أكثر خطأين يقع فيهما المبرمجون (حتى المحترفون منهم أحياناً) عند التعامل مع المؤشرات، وكيفية تصحيحهما.

الخطأ الأول: إسناد "قيمة" إلى المؤشر مباشرة (Value to Pointer)

المؤشر مُصمم ليأخذ "عنواناً" (Reference) فقط. لا يمكنك أبداً تخزين قيمة (رقم عادي) بداخله.

<div dir="ltr">

```cpp
int x = 10;
int *p;

// ❌ الخطأ الشائع:
p = x;
// الكومبايلر سيعطي خطأ: لا يمكنك تحويل 'int' إلى 'int*'
// لقد حاولت وضع قيمة المتغير x داخل المؤشر، وهذا مستحيل!

// ✅ التصحيح:
p = &x;
// يجب إعطاء المؤشر "عنوان" المتغير x باستخدام علامة &

```

</div>

الخطأ الثاني: إسناد "عنوان" إلى مفتاح القيمة (Reference to Dereferenced Pointer)

عندما نستخدم النجمة قبل المؤشر *p، فإننا نستخدم "المفتاح" للوصول إلى القيمة الموجودة داخل الصندوق. بالتالي، يجب إعطاء *p قيمة عادية وليس عنواناً!

<div dir="ltr">

```cpp
int x = 10;
int *p = &x;

// ❌ الخطأ الشائع:
*p = &x;
// الكومبايلر سيعطي خطأ: لا يمكنك وضع عنوان (Address) داخل قيمة (Value)
// المتغير *p ينتظر منك رقماً عادياً، وليس عنواناً جديداً!

// ✅ التصحيح:
*p = x;  // أو *p = 50;
// نقوم بإسناد القيمة الفعلية (رقم عادي)

```

</div>

💡 القاعدة الذهبية للمؤشرات (لا تنساها أبداً):

لكي تبرمج بدون أخطاء، احفظ هذه المعادلة البسيطة:

p (المؤشر بدون النجمة): يقبل & (عنوان/Reference) فـقـط!
(مثال: p = &x;)

*p (المؤشر مع النجمة/المفتاح): يقبل Value (قيمة عادية) فـقـط!
(مثال: *p = 50; أو \*p = x;)

## ⚖️ الدرس الثامن والثلاثون: المؤشرات مقابل المراجع (Pointers vs References)

سؤال هندسي شائع ومهم جداً: "طالما أن المراجع (References) تعطيني وصولاً لنفس الذاكرة، فلماذا تم اختراع المؤشرات (Pointers) من الأساس؟"

لفهم الفرق، يجب أن نعقد مقارنة بسيطة بينهما بناءً على سلوكهما في الذاكرة:

### 1. المرجع (Reference - اسم الدلع):

عندما نكتب int &x = a;، نحن لا نقوم بحجز مساحة جديدة في الذاكرة لـ x. المتغير x هو مجرد "اسم دلع" (Alias) للمتغير a. كلاهما يشير إلى نفس المكان، وليس للـ x وجود مادي منفصل في الذاكرة.

### 2. المؤشر (Pointer - الصندوق المستقل):

عندما نكتب int \*p = &a;، فإن نظام التشغيل يقوم بحجز "صندوق جديد" بالكامل في الذاكرة خاص بالمؤشر p، وهذا الصندوق له عنوانه الخاص المستقل، ويتم تخزين عنوان a كقيمة بداخله.

🃏 الفرق الجوهري (لماذا المؤشرات أقوى؟):

الفرق الحقيقي والخطير الذي يمنح المؤشرات قوتها (Flexibility) يكمن في وقت التشغيل (Runtime):

المرجع (Reference) يلتصق للأبد: بمجرد أن تجعل المرجع x يشير إلى a، لا يمكنك أبداً أن تغير رأيك وتجعله يشير إلى متغير آخر مثل b. لقد التصق به نهائياً! (محاولة تغييره ستؤدي لتغيير القيمة وليس تغيير الوجهة).

المؤشر (Pointer) هو بطاقة الجوكر: المؤشر متغير حر، يمكنك في أي لحظة أثناء تشغيل البرنامج أن تسحب المؤشر من a وتجعله يوجه إلى b، ثم إلى c.

💻 مقارنة عملية بالكود:

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

int main()
{
    int a = 10;
    int b = 20;

    // --- 1. المراجع (References) ---
    int &x = a; // x الآن هو اسم دلع لـ a

    // ❌ لا يمكنك توجيه المرجع x ليصبح اسم دلع لـ b!
    // الكود التالي سيقوم بنسخ قيمة b (وهي 20) ويضعها داخل a، ولكنه لن يغير المرجع!
    x = b;

    // --- 2. المؤشرات (Pointers) ---
    int *p = &a; // المؤشر p يؤشر حالياً على a

    // ✅ يمكنك بسهولة إعادة توجيه المؤشر في أي وقت!
    p = &b; // المؤشر p أصبح الآن يؤشر على b بكل سلاسة

    return 0;
}
```

</div>

💡 الخلاصة الهندسية:
نستخدم المراجع (References) عندما نحتاج طريقة آمنة وثابتة للوصول للمتغيرات (مثل تمرير المعاملات للدوال). ونستخدم المؤشرات (Pointers) عندما نحتاج لبناء هياكل بيانات ديناميكية معقدة (Dynamic Data Structures) تحتاج لتغيير وجهتها باستمرار أثناء عمل البرنامج (وهو ما ستدرسه لاحقاً في Data Structures).

## 🔄 الدرس التاسع والثلاثون: التمرير بالمرجع باستخدام المؤشرات (Call by Reference: Using Pointers)

لقد تعلمنا سابقاً كيفية تبديل قيمتين باستخدام دالة swap وتمرير المتغيرات كـ (References) بإضافة علامة & بجانب البارامترات: void swap(int &n1, int &n2).

الآن، وبما أننا فهمنا المؤشرات، يمكننا تحقيق نفس النتيجة تماماً باستخدام المؤشرات بدلاً من المراجع. هذا سيساعدنا على فهم كيف تعمل المراجع أصلاً خلف الكواليس.

### 1. تعديل دالة swap لتستقبل مؤشرات:

بدلاً من استقبال "أسماء دلع" (References)، سنجعل الدالة تستقبل مؤشرات (Pointers)، أي أنها تنتظر منا إرسال "عناوين" في الذاكرة.

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

// الدالة تستقبل مؤشرات (عناوين في الذاكرة)
void swap(int *n1, int *n2)
{
    int temp;

    // نستخدم "المفتاح" (*) للوصول للقيم الحقيقية داخل الصناديق وتبديلها
    temp = *n1;
    *n1 = *n2;
    *n2 = temp;
}

int main()
{
    int a = 1, b = 2;

    cout << "Before Swap: a = " << a << " | b = " << b << endl;

    // 🚨 تريكة هامة جداً عند الاستدعاء:
    // بما أن الدالة تطلب مؤشرات، يجب أن نرسل "العناوين" باستخدام علامة &
    swap(&a, &b);

    cout << "After Swap: a = " << a << " | b = " << b << endl;

    return 0;
}
```

</div>

### 2. لماذا نستخدم المؤشرات بدلاً من المراجع هنا؟

قد تسأل نفسك: "طالما أن النتيجة واحدة، لماذا أغلب نفسي باستخدام المؤشرات وأرسل عناوين (&) وأستخدم مفاتيح (\*)، بدلاً من استخدام المراجع (&) السهلة والمباشرة؟"

الجواب الهندسي: في هذا المثال البسيط (swap)، استخدام المراجع هو الخيار الأسهل والأفضل فعلاً.
لكن في مشاريع معقدة، قد تحتاج داخل الدالة نفسها إلى تغيير اتجاه المؤشر n1 ليؤشر على متغير ثالث n3 بناءً على شرط معين. كما ذكرنا في الدرس السابق: المراجع لا يمكن تغيير وجهتها في وقت التشغيل (Runtime)، بينما المؤشرات تمنحك هذه المرونة الكاملة.

💡 ملخص قاعدة الاستدعاء:
إذا كان بارامتر الدالة مؤشراً void Function(int \*p)، فعند استدعاء الدالة من الـ main يجب أن تمرر لها عنوان المتغير هكذا: Function(&a).

## 🗄️ الدرس الأربعون: المؤشرات والمصفوفات (Pointers and Arrays)

كما أشرنا سابقاً، المؤشر يمكنه التأشير على أي شيء موجود في الذاكرة، بما في ذلك المصفوفات (Arrays).
المصفوفة في C++ ما هي إلا مجموعة من المتغيرات من نفس النوع متراصة بشكل متتالي في الذاكرة. ولكل عنصر في هذه المصفوفة عنوان (Address) خاص به.

### 1. ربط المؤشر بالمصفوفة:

عندما نقوم بتعريف مصفوفة arr ونريد لمؤشر ptr أن يؤشر عليها، لا نحتاج لاستخدام علامة & مع اسم المصفوفة مباشرة؛ بل نساوي المؤشر باسم المصفوفة، وحينها سيقوم المؤشر تلقائياً بـ التأشير على أول عنصر في المصفوفة.

<div dir="ltr">

```cpp
int arr[4] = {10, 20, 30, 40};
int *ptr;

// المؤشر الآن يؤشر على أول عنصر في المصفوفة: arr[0]
ptr = arr;

```

</div>

### 2. حساب المؤشرات (Pointer Arithmetic):

بما أن عناصر المصفوفة متتالية في الذاكرة، والمؤشر يقف الآن عند العنصر الأول، ماذا يحدث لو قمنا بجمع رقم على المؤشر (مثال: ptr + 1)؟
لن يقوم الكومبايلر بجمع الرقم 1 رياضياً، بل سينتقل للمؤشر للـ "المكان (العنوان) التالي" في الذاكرة، أي أنه سيؤشر على العنصر الثاني arr[1]، وهكذا!

ptr : يشير إلى عنوان العنصر الأول arr[0]

ptr + 1 : يشير إلى عنوان العنصر الثاني arr[1]

ptr + 2 : يشير إلى عنوان العنصر الثالث arr[2]

### 3. استخراج القيم (Dereferencing):

للحصول على القيم الموجودة داخل هذه العناوين، نستخدم "المفتاح" \* مع وضع العملية الحسابية بين أقواس هكذا:

\*(ptr) : يجلب قيمة العنصر الأول.

\*(ptr + 1) : يجلب قيمة العنصر الثاني.

💻 تطبيق عملي (طباعة المصفوفة باستخدام المؤشرات وحلقة التكرار):

بدلاً من كتابتها سطراً بسطر كما طلب الدكتور كتمرين لك، يمكننا المرور على عناصر المصفوفة وطباعة عناوينها وقيمها باستخدام حلقة for بسيطة جداً مع المؤشرات:

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

int main()
{
    int arr[4] = {10, 20, 30, 40};
    int *ptr = arr; // المؤشر يؤشر على أول عنصر في المصفوفة

    cout << "--- Printing Array Using Pointers ---\n\n";

    // استخدام for loop مع حساب المؤشرات (Pointer Arithmetic)
    for (int i = 0; i < 4; i++)
    {
        // (ptr + i)   -> يطبع عنوان الذاكرة
        // *(ptr + i)  -> يطبع القيمة الموجودة داخل العنوان
        cout << "Address: " << ptr + i << " | Value: " << *(ptr + i) << endl;
    }

    return 0;
}

```

</div>

💡 الفائدة الهندسية:
التعامل مع المصفوفات باستخدام المؤشرات يمنحك سرعة فائقة جداً (خصيصاً في لغات مثل C و C++)، وهو المفهوم الأساسي الذي تعتمد عليه المصفوفات في الخلفية؛ فاسم المصفوفة arr هو في الحقيقة يعمل كـ "مؤشر ثابت" يشير لأول عنصر!

## 🏗️ الدرس الحادي والأربعون: المؤشرات والهياكل (Pointers and Structure)

كما رأينا أن المؤشر يمكنه التأشير على أنواع بيانات بسيطة كـ int و float ومصفوفات، فإنه قادر أيضاً على التأشير على أنواع بيانات مركبة صممناها بأنفسنا مثل الهياكل (Structures).

### 1. ربط المؤشر بالهيكل:

العملية لا تختلف أبداً عن ربط المؤشر بمتغير عادي، كل ما نحتاجه هو:

تعريف الهيكل (مثلاً stEmployee).

إنشاء متغير من نوع هذا الهيكل وإعطاؤه قيم.

تعريف مؤشر من نفس نوع الهيكل، وتخزين عنوان المتغير فيه.

### 2. التريكة الهندسية: معامل السهم (Arrow Operator ->):

للوصول إلى عناصر المتغير العادي (مثل الاسم أو الراتب)، كنا نستخدم النقطة . (مثال: emp1.Name).
ولكن، عندما نتعامل مع مؤشر يؤشر على هيكل، لا يمكننا استخدام النقطة مباشرة.
لدينا طريقتان للوصول للمتغيرات عن طريق المؤشر:

الطريقة التقليدية (المعقدة): أن نستخدم مفتاح المؤشر بين أقواس ثم نقطة (\*ptr).Name وهذا الشكل مزعج في القراءة.

طريقة المحترفين (Arrow Operator): وفرت لنا C++ معامل السهم -> كاختصار أنيق جداً. كل ما عليك كتابته هو ptr->Name.
(ملاحظة: بيئات التطوير الذكية IDEs مثل Visual Studio تقوم تلقائياً بتحويل النقطة . إلى سهم -> بمجرد أن تكتشف أنك تتعامل مع مؤشر!)

💻 تطبيق عملي بالكود:

<div dir="ltr">

```cpp
#include <iostream>
#include <string>
using namespace std;

// تعريف الهيكل
struct stEmployee
{
    string Name;
    float Salary;
};

int main()
{
    // 1. إنشاء المتغير وإعطاؤه قيم
    stEmployee emp1;
    emp1.Name = "Mohammed Abu-Hadhoud";
    emp1.Salary = 5000;

    // 2. إنشاء المؤشر وجعله يؤشر على المتغير emp1
    stEmployee *ptr = &emp1;

    // --- الطباعة بالطريقة العادية ---
    cout << "--- Accessing via Variable ---" << endl;
    cout << "Name: " << emp1.Name << endl;
    cout << "Salary: " << emp1.Salary << endl;

    // --- الطباعة باستخدام المؤشر (معامل السهم ->) ---
    cout << "\n--- Accessing via Pointer (Arrow Operator) ---" << endl;
    cout << "Name: " << ptr->Name << endl;
    cout << "Salary: " << ptr->Salary << endl;

    return 0;
}

```

</div>

💡 الخلاصة: عندما تريد الوصول للخصائص (Members) الخاصة بـ Struct أو Class باستخدام المؤشرات، استخدم دائماً السهم -> بدلاً من النقطة ..

## 🌌 الدرس الثاني والأربعون: مؤشر الفراغ (Pointer to Void)

كما تعلمنا سابقاً، المؤشر شديد الحساسية لنوع البيانات. فإذا قمت بتعريف مؤشر من نوع int \*، فإنه لا يمكن أن يؤشر إلا على متغيرات من نوع int. وإذا حاولت جعله يؤشر على متغير float، سيغضب الكومبايلر ويرفض ذلك ويعتبره خطأً برمجياً.

ولكن، ماذا لو أردت بناء مؤشر "عام" (Generic) يمكنه التأشير على أي نوع من البيانات في الذاكرة وتغيير وجهته لأنواع مختلفة أثناء تشغيل البرنامج (Runtime)؟ هنا يظهر دور مؤشر الفراغ (void \*).

### 1. ما هو الـ Void Pointer؟

هو مؤشر لا يمتلك نوعاً محدداً مسبقاً. يمكنه تخزين عنوان لأي متغير (سواء كان int، float، double، struct، إلخ) دون أي اعتراض من الكومبايلر.

### 2. المشكلة: كيف نفك المرجعية (Dereferencing)؟

بما أن المؤشر لا يعرف نوع البيانات التي يؤشر عليها، فإنه لا يعرف حجمها في الذاكرة (هل يقرأ 4 بايت لأنه Integer أم يقرأ 8 بايت لأنه Double؟). لذلك، لا يمكنك استخدام المفتاح * مباشرة لطباعة القيمة!
مثال خاطئ: cout << *ptr; سيعطيك خطأ لأن الكومبايلر لا يعرف نوع القيمة.

### 3. الحل: التحويل (Static Cast)

لقراءة القيمة الموجودة داخل العنوان، يجب أولاً أن تخبر الكومبايلر بالنوع الحقيقي للمتغير الذي تقرأه الآن باستخدام أداة التحويل static_cast<Type*>(ptr)، ثم تضع النجمة * في الخارج لفك المرجعية وجلب القيمة.

💻 تطبيق عملي بالكود:

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

int main()
{
    // تعريف مؤشر عام (لا نوع له)
    void *ptr;

    float f1 = 10.5;
    int x = 50;

    // --- 1. التأشير على Float ---
    ptr = &f1; // المؤشر الآن يؤشر على float (بدون أخطاء)

    cout << "Address of f1: " << ptr << endl;

    // جلب القيمة: يجب تحويل المؤشر إلى float* أولاً لتتمكن من استخدام النجمة *
    cout << "Value of f1: " << *(static_cast<float*>(ptr)) << endl;


    // --- 2. تغيير الوجهة إلى Int ---
    ptr = &x; // المؤشر غير وجهته وأصبح يؤشر على int بسلاسة

    cout << "\nAddress of x: " << ptr << endl;

    // جلب القيمة: يجب تحويل المؤشر إلى int* أولاً
    cout << "Value of x: " << *(static_cast<int*>(ptr)) << endl;

    return 0;
}
```

</div>

💡 الفائدة الهندسية:
نحتاج لـ void pointers في المواقف المتقدمة جداً، خاصة عندما نكتب دوالاً (Functions) يجب أن تتعامل مع أنواع بيانات غير معروفة مسبقاً، فهي تعطينا مرونة فائقة (Flexibility) لاستقبال أي نوع بيانات ثم تحويله (Casting) للنوع المطلوب في وقت التشغيل.
