# الفصل الثاني: أدوات متقدمة وتنظيم الكود (Miscellaneous & Functions)

## 📚 الدرس الثامن: إنشاء مكتبتك الخاصة (Create Your Own Library)

مع كثرة المشاريع، ستلاحظ أن هناك دوال (Functions) تستخدمها دائماً (مثل دوال قراءة الأرقام، التحقق من المدخلات، أو توليد أرقام عشوائية). بدلاً من إعادة كتابتها في كل مشروع، المبرمج المحترف يضعها في "مكتبة" (Library) خاصة به ليعيد استخدامها لاحقاً (Reusability).### خطوات إنشاء المكتبة (Local Library):
    1. في (Solution Explorer)، اضغط كليك يمين على مجلد (Header Files).
    2. اختر `Add -> New Item -> Header File (.h)`.
    3. قم بتسميتها (مثال: `MyLibrary.h`).
    4. **تنظيم المكتبة (Namespace):** لمنع تداخل الأسماء، نضع دوال المكتبة تحت مظلة (Namespace) خاص بنا.
**شكل الكود داخل ملف المكتبة (`MyLibrary.h`):**

</div>

<div dir="ltr">

```cpp
#pragma once
#include <iostream>
using namespace std;

namespace MyLib
{
    void TestFunction()
    {
        cout << "Hello from My Library!\n";
    }
}
```

</div>

<div dir="rtl">


كيف تستدعيها في مشروعك الرئيسي؟

نستخدم علامات التنصيص "" بدلاً من الأقواس <> لأنها مكتبة محلية وليست من أصل اللغة: #include "MyLibrary.h".
لاستخدام الدالة نكتب: MyLib::TestFunction();.
نصيحة: تجنب كتابة using namespace MyLib; في المشاريع الكبيرة حتى لا تختلط أسماء دوال مكتبتك بدوال مكتبات أخرى.

## ⚡ الدرس التاسع: المعامل الثلاثي (Ternary Operator / Short Hand IF)
هو طريقة مختصرة وأنيقة لكتابة جملة if...else البسيطة في سطر واحد فقط.

القاعدة (Syntax): Condition ? True_Expression : False_Expression
مقارنة بالكود:

</div>

<div dir="ltr">

```cpp
// الطريقة التقليدية
if (mark >= 50)
    result = "Pass";
else
    result = "Fail";

// طريقة المحترفين (Ternary Operator)
result = (mark >= 50) ? "Pass" : "Fail";

```

</div>

<div dir="rtl">


المميزات: يمكنك استخدامه مباشرة داخل جملة cout، أو وضع معامل ثلاثي آخر بداخله (Nested Ternary) لاختبار شروط متعددة (مثل التحقق هل الرقم صفر، أم موجب، أم سالب في سطر واحد).

## 🔄 الدرس العاشر: حلقة التكرار المخصصة للمجموعات (Ranged Loop)
دالة For Loop التقليدية تتطلب تعريف عداد (int i = 0) وتحديد شرط التوقف، ومقدار الزيادة. أما الـ (Ranged Loop) فهي مصممة خصيصاً للمرور على "المجموعات" (Collections) مثل المصفوفات (Arrays) بطريقة أسهل بكثير.

</div>

<div dir="ltr">

```cpp
int numbers[] = {10, 20, 30, 40, 50};

// Ranged Loop
for (int n : numbers)
{
    cout << n << endl;
}
```

</div>

<div dir="rtl">

كيف تعمل؟ تأخذ عنصراً من المصفوفة numbers وتضعه في المتغير n وتطبعه، ثم تذهب للعنصر الذي يليه تلقائياً حتى تنتهي المصفوفة دون الحاجة لمعرفة طولها.

## 🛡️ الدرس الحادي عشر: التحقق من صحة الأرقام (Validate Number)

المشكلة: ماذا يحدث لو طلب برنامجك إدخال رقم int، وقام المستخدم بإدخال حرف مثل A؟
سيدخل البرنامج في حالة انهيار أو تكرار لا نهائي (Infinite Loop) ولن تتمكن من إيقافه. لحل هذه المشكلة نستخدم أدوات التحقق المدمجة في كائن الإدخال cin.

أدوات الحل الثلاثة:

1.cin.fail():
دالة ترجع True إذا فشل الإدخال (بسبب إدخال نوع بيانات خاطئ). نستخدمها كشرط داخل حلقة while لاكتشاف الخطأ.

2.cin.clear():
عندما يفشل الإدخال، يتعطل cin تماماً. هذه الدالة تمسح "حالة الخطأ" ليعود كائن cin للعمل والاستقبال من جديد.

3.cin.ignore(10000, '\n'):
تتجاهل (وتمسح) باقي الأحرف الخاطئة العالقة في الذاكرة المؤقتة (Buffer) حتى تصل لزر الإدخال (Enter)، لتنظيف الذاكرة تماماً للمحاولة القادمة.

الكود العملي لحل المشكلة:

</div>

<div dir="ltr">

```cpp

int number;
cout << "Enter a number: ";
cin >> number;

while (cin.fail()) 
{
    cin.clear(); // مسح الخطأ
    cin.ignore(10000, '\n'); // تفريغ الذاكرة
    cout << "Invalid Input! Please enter a valid number: ";
    cin >> number;
}
```

</div>

## 🔢 الدرس الثاني عشر والثالث عشر: معاملات البت (Bitwise AND & OR)

هناك فرق شاسع بين المعاملات المنطقية العادية (Logical) وبين معاملات مستوى البت (Bitwise) التي تنزل لأعمق مستوى في الكمبيوتر وتتعامل مع لغة الآلة (الأصفار والآحاد).

1. معامل (Bitwise AND - &) إشارة واحدة:

يقوم بتحويل الأرقام إلى (Binary)، ويقارن كل "بت" بما يقابله.

القاعدة: إذا كان الاثنان 1 يخرج 1، أي شيء آخر يخرج 0.

مثال: cout << (12 & 25);

النتيجة: ستكون 8. (لأنه تم تحويل الـ 12 والـ 25 لباينري، وتم عمل AND لكل خانة منفردة، والناتج الباينري تمت إعادته لرقم عشري).

ملاحظة للفرق: المعامل المنطقي العادي (12 && 25) نتيجته ستكون 1 (True) لأن كلاهما رقم يمثل قيمة صحيحة.

2. معامل (Bitwise OR - |) إشارة واحدة:

يحول الأرقام لـ (Binary) ويقارن كل "بت" بما يقابله.

القاعدة: يكفي أن يكون أحدهما 1 ليخرج النتيجة 1.

مثال: cout << (12 | 25);

النتيجة: ستكون 29.

(ملاحظة هندسية: هذه المعاملات تبدو غريبة ولا معنى لها الآن، ولكن لها تطبيقات سحرية متقدمة جداً في تسريع العمليات الحسابية، التشفير، وبرمجة الأنظمة المدمجة Embedded Systems لأنها تتعامل مباشرة مع المعالج).
