# الفصل العاشر: الحجز الديناميكي للذاكرة (Dynamic Memory Allocation)

## 🧠 الدرس الثالث والأربعون: إدارة الذاكرة باستخدام new و delete

طوال الدروس الماضية في فصل المؤشرات (Pointers)، ربما سألت نفسك: "لماذا أُتعب نفسي بتعريف مؤشر ليؤشر على متغير موجود أصلاً؟ ما الفائدة الحقيقية من كل هذا؟"
الإجابة تكمن في موضوع اليوم: الحجز الديناميكي للذاكرة. بدون المؤشرات، لا يوجد حجز ديناميكي!

### 1. ما هو الحجز الديناميكي للذاكرة؟

الطريقة التقليدية (Static Allocation): عندما تُعرف متغيراً بالطريقة العادية int x = 50;، يتم حجز مساحة له في الذاكرة بمجرد تشغيل البرنامج، وتظل هذه المساحة محجوزة ومستهلكة طوال فترة عمل البرنامج (أو الدالة) حتى وإن لم تعد بحاجة إليها.

الطريقة الديناميكية (Dynamic Allocation): تمنحك لغة C++ القدرة على إنشاء المتغيرات أثناء تشغيل البرنامج (Runtime) (مثلاً: داخل شرط if معين إذا تحقق فقط)، والأهم من ذلك: القدرة على مسحها من الذاكرة فور الانتهاء منها لتوفير مساحة لمهام أخرى.

### 2. الكلمات المحجوزة new و delete:

تتم عملية إدارة الذاكرة برمجياً باستخدام كلمتين سحريتين:

new: تطلب من نظام التشغيل حجز مساحة جديدة في الذاكرة لنوع معين (مثلاً int). نظام التشغيل يرد بإعطائك "عنوان" (Address) هذه المساحة الجديدة. وبما أنه عنوان، فإننا نحتاج إلى مؤشر (Pointer) لالتقاطه وتخزينه!

delete: تقوم بتحرير (مسح) المساحة التي يؤشر عليها المؤشر وإعادتها لنظام التشغيل ليتم استخدامها في مهام أخرى.

💻 تطبيق عملي بالكود:

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

int main()
{
    // 1. تعريف المؤشرات (هي مجرد "طرف خيط" لا تستهلك مساحة للبيانات بعد)
    int *ptrX;
    float *ptrY;

    // 2. الحجز الديناميكي في وقت التشغيل (Runtime)
    // اطلب من النظام مساحة لـ int ومساحة لـ float، وخزن عناوينهما في المؤشرات
    ptrX = new int;
    ptrY = new float;

    // 3. إسناد القيم باستخدام "المفتاح" (*)
    *ptrX = 45;
    *ptrY = 58.35f;

    // 4. استخدام المتغيرات وطباعتها
    cout << "Value of dynamically allocated int: " << *ptrX << endl;
    cout << "Value of dynamically allocated float: " << *ptrY << endl;

    // 5. 🚨 الخطوة الأهم: تحرير الذاكرة (Memory Deallocation)
    // بمجرد الانتهاء، يجب أن نحذف المساحات التي حجزناها لكي لا يمتلئ الرام
    delete ptrX;
    delete ptrY;

    return 0;
}

```

</div>

💡 القاعدة الذهبية من المهندس أبو هدهود:
"كن كريماً في كل حياتك، إلا في البرمجة! يجب أن تكون بخيلاً جداً في الذاكرة."
القاعدة البرمجية الصارمة تقول: كل new تكتبها في الكود، يجب أن يقابلها delete. إذا حجزت ذاكرة ونسيت استخدام delete، سيؤدي ذلك لمشكلة خطيرة تُسمى (تسرب الذاكرة - Memory Leak)، حيث يظل البرنامج يستهلك الرام بمرور الوقت حتى ينهار النظام بأكمله!

## 📏 الدرس الرابع والأربعون: المصفوفات الديناميكية (Dynamic Arrays: new and delete)

في الدروس السابقة (تحديداً في فصل المصفوفات)، تعلمنا أن المصفوفة العادية (Static Array) يجب تحديد حجمها قبل تشغيل البرنامج، مثل: float arr[100];.
هذا الأسلوب يطرح مشكلتين كارثيتين:

هدر الذاكرة (Memory Waste): إذا أردت تخزين درجات 3 طلاب فقط، فأنت تهدر مساحة 97 طالباً في الذاكرة بلا أي فائدة!

انهيار البرنامج (Crash): إذا أردت إدخال بيانات 101 طالب، فالبرنامج سينهار لعدم وجود مساحة كافية.

هنا يبرز السحر الحقيقي للمؤشرات (Pointers) والحجز الديناميكي، حيث سنتعلم كيف نُنشئ مصفوفة حجمها يتحدد بناءً على طلب المستخدم أثناء تشغيل البرنامج (Runtime) وبدون أي هدر للذاكرة!

### 1. طريقة حجز المصفوفة الديناميكية:

بدلاً من تعريف المصفوفة بحجم ثابت، نسأل المستخدم عن العدد الذي يريده، ثم نستخدم new متبوعة بنوع البيانات وأقواس المصفوفة [ ].

<div dir="ltr">

```cpp
int NumberOfStudents;
cout << "Enter total number of students: ";
cin >> NumberOfStudents;

// حجز المصفوفة ديناميكياً بناءً على إدخال المستخدم
float *ptr = new float[NumberOfStudents];
```

</div>

ما حدث هنا هو أن الكومبايلر حجز مصفوفة في الذاكرة بحجم NumberOfStudents بالضبط (سواء كان 3 أو مليون)، وجعل المؤشر ptr "طرف الخيط" الذي يؤشر على أول عنصر فيها.

### 2. طريقة الحذف (Deallocation) 🚨:

عندما ننتهي من المصفوفة الديناميكية، يجب أن نحذفها. ولكن احذر! لا تستخدم delete ptr; العادية، بل يجب أن تضع أقواس المصفوفة المربعة لإخبار الكومبايلر أن يحذف السلسلة بالكامل وليس العنصر الأول فقط.
الشكل الصحيح للحذف: delete[] ptr;

💻 تطبيق عملي كامل (إدخال وطباعة درجات الطلاب):

<div dir="ltr">

```cpp
#include <iostream>
using namespace std;

int main()
{
    int num;
    cout << "Enter total number of students: ";
    cin >> num;

    // 1. حجز الذاكرة الديناميكي
    float *ptr = new float[num];

    // 2. إدخال العلامات (باستخدام المؤشرات)
    cout << "\n--- Entering Grades ---\n";
    for (int i = 0; i < num; i++)
    {
        cout << "Student " << i + 1 << " grade: ";
        cin >> *(ptr + i); // استخدام حساب المؤشرات للوصول لكل خلية
    }

    // 3. طباعة العلامات
    cout << "\n--- Displaying Grades ---\n";
    for (int i = 0; i < num; i++)
    {
        cout << "Student " << i + 1 << ": " << *(ptr + i) << endl;
    }

    // 4. تحرير الذاكرة (إياك أن تنساها!)
    delete[] ptr;

    return 0;
}

```

</div>

💡 الفائدة الهندسية:
من خلال هذا الأسلوب، أنت حجزت الذاكرة بمقدار الحاجة (مثلاً 3 طلاب)، وفور طباعة الدرجات، قمت بمسحها تماماً من الذاكرة (عبر delete[]). هكذا يكون البرنامج أسرع بأضعاف المرات، والموارد متاحة دائماً، وهو ما يميز لغة C++ عن باقي اللغات التي تعتمد على (Garbage Collectors) وتقوم بهذه العمليات في الخلفية ببطء.

### 🚨 ماذا يحدث إذا نسينا استخدام delete؟ (تسرب الذاكرة - Memory Leak)

إذا قمت بحجز مساحة في الذاكرة باستخدام new، فإن نظام التشغيل يسجل أن هذه المساحة "مشغولة". إذا انتهت الدالة (Function) أو قمت بتغيير اتجاه المؤشر دون أن تستخدم delete، سيتم تدمير المؤشر (لأنه متغير محلي)، ولكن الذاكرة المحجوزة ستظل مسجلة كمشغولة للأبد!

لقد فقدت "المفتاح/العنوان" الذي يوصلك لهذه الذاكرة، ولم يعد بإمكانك مسحها، ولن يستطيع نظام التشغيل استخدامها لبرامج أخرى. هذه الكارثة تُعرف بـ (تسرب الذاكرة - Memory Leak).

💻 مثال عملي لكارثة تسرب الذاكرة:

<div dir="ltr">

```cpp

#include <iostream>
using namespace std;

void CreateLeak()
{
    // نحجز مصفوفة ضخمة جداً (مثلاً مليون عنصر)
    int *ptr = new int[1000000];

    // الدالة انتهت هنا!
    // المؤشر ptr سيتم تدميره (لأنه Local Variable)
    // لكن الـ 1000000 عنصر ستظل محجوزة في الرام لأننا نسينا كتابة delete[] ptr;
}

int main()
{
    // حلقة تكرار لا نهائية تستدعي الدالة
    while (true)
    {
        CreateLeak();
    }

    return 0;
}

```

</div>

ماذا سيحدث عند تشغيل هذا الكود؟
مع كل لفة في الـ while، سيقوم البرنامج باقتطاع جزء ضخم من الـ RAM ولا يعيده أبداً. في غضون ثوانٍ معدودة، سيستهلك البرنامج كل سعة الذاكرة العشوائية المتاحة في جهازك، وسينهار البرنامج بالكامل (Crash) مع رسالة خطأ (Out of Memory)، بل وقد يتسبب في تجميد نظام التشغيل (Windows/Mac) بأكمله!

💡 الفائدة الهندسية:
من خلال هذا الأسلوب المنضبط، أنت تحجز الذاكرة بمقدار الحاجة (مثلاً 3 طلاب)، وفور طباعة الدرجات، تقوم بمسحها تماماً من الذاكرة (عبر delete[]). هكذا يكون البرنامج أسرع بأضعاف المرات، والموارد متاحة دائماً، وهو ما يميز لغة C++ عن باقي اللغات التي تعتمد على (Garbage Collectors) وتقوم بهذه العمليات في الخلفية ببطء.

## 🏗️ الدرس الخامس والأربعون: الذاكرة (Stack vs Heap)

الآن، بعد أن تعمقنا في المؤشرات والحجز الديناميكي، حان الوقت لفهم "أين تذهب هذه البيانات فعلياً؟".
لفهم ذلك، يجب أن نعلم أن الذاكرة العشوائية (RAM) تُقسم افتراضياً لأي برنامج يعمل إلى أربعة أقسام رئيسية:

### 1. أقسام الذاكرة الأربعة للبرنامج:

قسم الأوامر (Instructions / Code Segment):

مساحة صغيرة جداً يُخزن فيها الكود المترجم (Compiled Code) الخاص ببرنامجك ليقوم المعالج بتنفيذه.

قسم المتغيرات العامة والساكنة (Global & Static Variables):

مساحة مخصصة للمتغيرات التي قمت بتعريفها كـ Global (خارج كل الدوال) أو Static. هذه المتغيرات يتم حجزها مرة واحدة وتظل موجودة طوال فترة حياة البرنامج.

الـ Stack (المكدس):

هو المساحة التي يخصصها نظام التشغيل لبرنامجك بناءً على المتغيرات المحلية (Local Variables) والمصفوفات الثابتة والدوال التي كتبتها في الكود قبل الـ Runtime.

مساحته محدودة وثابتة: (مثلاً يعطيك النظام 1 ميجابايت كحد أقصى).

ماذا يُخزن فيه؟ يتم تخزين أي متغير عادي تعرفه داخل دالة (مثل int x = 5;)، وكذلك يتم تخزين المؤشرات نفسها هنا (المؤشر كمتغير يعيش في الـ Stack).

الـ Heap (الكومة):

هو بنك الذاكرة المفتوح! هو المساحة الشاسعة المتبقية من الـ RAM في جهازك (مثلاً بقية الـ 16 جيجابايت).

هذه المساحة ليست مخصصة لبرنامجك مسبقاً، بل يجب أن "تطلبها" صراحةً متى احتجتها.

### 2. التشبيه الهندسي العبقري (المصروف ودفتر الشيكات):

لتبسيط الفكرة تماماً، تخيل السيناريو التالي:

الـ Stack (المصروف المحدود): قبل أن تبدأ رحلتك (قبل تشغيل البرنامج)، قام والدك (نظام التشغيل) بإعطائك مصروفاً محدوداً في جيبك (مثلاً 100 دولار). هذا هو الـ Stack. أنت مقيد بهذا المبلغ، وإذا حاولت حجز مصفوفة ضخمة جداً تتعدى ميزانيتك، ستتعرض للإفلاس والانهيار (Stack Overflow).

الـ Heap (البنك): البنك يحتوي على ملايين الدولارات (مساحة ضخمة جداً)، ولكنه بعيد عن جيبك، ولا يمكنك أخذ أي مبلغ منه مباشرة بيدك.

الـ Pointer (دفتر الشيكات): كيف إذن تسحب من البنك؟ ستحتاج إلى "دفتر شيكات". المؤشر الذي تعرفه في كودك (ويعيش في جيبك/الـ Stack) هو بمثابة دفتر الشيكات.

عندما تكتب الكود التالي:
int \*ptr = new int;
أنت حرفياً تستخرج دفتر الشيكات (Pointer)، وتكتب شيكاً باستخدام الكلمة السحرية (new) وتطلب من البنك (Heap) أن يفتح لك رصيداً جديداً. البنك يعطيك الرصيد ويرد لك "العنوان/الرقم السري" لتسجله في دفتر شيكاتك.

💡 الخلاصة الفنية:

الخلاصة هي أن المؤشرات (Pointers) هي "المفتاح أو المهرب" الوحيد الذي يسمح لك بتجاوز المساحة المحدودة المخصصة لبرنامجك (Stack)، لتنطلق وتحجز مساحات هائلة ديناميكياً من البنك المفتوح للذاكرة (Heap) أثناء تشغيل البرنامج (Runtime). ولهذا السبب بالتحديد، لغة C++ سريعة وجبارة جداً.
